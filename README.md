# 📘 Day 1 - Git Practice Assignment

## 📌 Objective

The goal of this assignment was to understand and practice the fundamental Git operations that are used in day-to-day software development and DevOps workflows.

---

# 📚 Topics Covered

* 📁 Repository
* 📥 Clone
* ➕ Add
* 💾 Commit
* 📤 Push & Pull
* 🔀 Branch

---

# 🛠️ Environment

* **Version Control:** Git
* **Remote Repository:** GitHub
* **Operating System:** Amazon Linux

---

# ✅ Tasks Performed

## 1. Created a Local Git Repository

* Created a new project directory.
* Initialized it as a Git repository.
* Created the following structure:

```text
git-learning/
│
├── README.md
└── notes/
    ├── linux.txt

```

---

## 2. Checked Repository Status

Verified the repository state using Git status to understand:

* Untracked files
* Staged files
* Working directory changes

---

## 3. Staged Files

Practiced staging:

* Individual files
* Multiple files
* Entire project

This helped understand the difference between the **Working Directory** and the **Staging Area**.

---

## 4. Created Commits

Created meaningful commits after staging changes.

Learned that:

* Every commit is a snapshot of the project.
* Each commit has a unique commit ID.
* Good commit messages improve project history.

---

## 5. Connected to GitHub

* Created a remote repository.
* Linked the local repository with GitHub.
* Verified the remote configuration.
* Successfully pushed the project.

---

## 6. Cloned the Repository

Cloned the GitHub repository into a different directory.

Verified that:

* All project files were downloaded.
* Commit history was preserved.
* Remote (`origin`) was automatically configured.

---

## 7. Created a Feature Branch

Created a new branch:

```text
feature-docker
```

Performed the following:

* Switched to the branch.
* Added a new file.
* Committed the changes.
* Pushed the branch to GitHub.

---

## 8. Practiced Pull

Made changes directly on GitHub and then updated the local repository using **Pull**.

Verified that:

* The latest remote changes were successfully reflected in the local repository.

---

# 📂 Git Workflow Practiced

```text
Working Directory
        │
        ▼
   git add
        │
        ▼
 Staging Area
        │
        ▼
  git commit
        │
        ▼
 Local Repository
        │
        ▼
   git push
        │
        ▼
 GitHub Repository
```

---

# 📖 Key Concepts Learned

* What a Git repository is.
* Difference between Local and Remote repositories.
* Purpose of the Staging Area.
* Creating meaningful commits.
* Uploading changes using Push.
* Downloading updates using Pull.
* Cloning repositories.
* Working with feature branches.
* Basic Git collaboration workflow.
